## 问题列表

### ISSUE-001 执行页缺少学习资料参考区
- 来源：执行页体验反馈
- 描述：当前执行页包含标题、任务区、工作区、编码区、运行结果区、状态分析，但缺少“学习资料参考区”，难以支持一轮完整学习活动。
- 分类：产品逻辑问题
- 是否影响当前 MVP 演示：是
- 优先级：P0
- 建议方案：在执行页补一个轻量“参考区/知识上下文卡片”，承载题目提示、知识标签、思路框架或参考资料入口；不重做整体架构。
- 影响页面/模块：
  - `frontend/src/components/ExecutionPage.tsx`
  - `frontend/src/components/TaskBrief.tsx`
- 建议负责 agent：`Agent D`
- 当前处理结论：已完成 P0 热修。执行页已补“学习参考区”，前端通过 `reference.*` 轻量事件记录使用行为；daemon 侧继续保持兼容接收、暂不纳入主推理。详见 `docs/热修记录-reference-zone.md`。

### ISSUE-002 执行页整体视觉 AI 味过重
- 来源：执行页体验反馈
- 描述：界面风格偏“AI 产品”，不够像自然学习工作台。
- 分类：交互问题
- 是否影响当前 MVP 演示：是
- 优先级：P1
- 建议方案：做轻量去 AI 味改造，调整文案语气、视觉层次、状态卡片表达，不做整站重设计。
- 影响页面/模块：
  - `frontend/src/components/StatusPanel.tsx`
  - `frontend/src/components/ExecutionPage.tsx`
  - `frontend/src/styles.css`
- 建议负责 agent：`Agent D`，如需视觉稿可辅助 `UI/UX Agent`
- 当前处理结论：记录，不进入热修。

### ISSUE-003 执行页信息架构需要重设计
- 来源：执行页体验反馈
- 描述：希望架构 Agent 重新设计执行页结构。
- 分类：新需求
- 是否影响当前 MVP 演示：否
- 优先级：P2

- 建议负责 agent：后续单独指派 `架构 Agent`
- 当前处理结论：记录，不进入热修。

### ISSUE-004 复盘页信息不易理解
- 来源：复盘页体验反馈
- 描述：当前复盘页每条信息含义不够清楚，用户不容易快速读懂。
- 分类：交互问题 / 文案问题
- 是否影响当前 MVP 演示：是
- 优先级：P0
- 建议方案：按“先结论、后证据、再建议”重排；补中文解释、术语释义、区块导语。
- 影响页面/模块：
  - `frontend/src/components/SessionReportPage.tsx`
  - `frontend/src/utils/report.ts`
- 建议负责 agent：`Agent E`
- 当前处理结论：已进入 P0 热修，页面已重排为“先结论、后证据、再建议”，并补状态释义、区块导语、学习化文案与稀疏数据兜底。

### ISSUE-005 复盘生成过程缺少学习流程感与科学流程说明
- 来源：复盘页体验反馈
- 描述：复盘生成过快，用户感知不到“它是如何基于学习过程形成”的。
- 分类：产品逻辑问题
- 是否影响当前 MVP 演示：是
- 优先级：P0
- 建议方案：补“复盘如何生成”说明区，明确事件采集、特征快照、状态推理、报告聚合四步；不引入复杂异步生成流程。
- 影响页面/模块：
  - `frontend/src/components/SessionReportPage.tsx`
  - 可选补充文案来自 `daemon/app/services/state_vector_service.py`
- 建议负责 agent：`Agent E`，必要时 `Agent C` 提供模型说明素材
- 当前处理结论：已进入 P0 热修，复盘页新增“复盘如何生成”四步说明区，解释数据来源、状态推理路径与“本地即时聚合所以很快”的原因，并与结论区、证据区形成清晰阅读顺序。

### ISSUE-006 算法建模的科学性展示不足
- 来源：体验反馈
- 描述：当前缺少清晰方式向用户/评委解释算法建模为什么成立、为什么可信。
- 分类：产品逻辑问题
- 是否影响当前 MVP 演示：是
- 优先级：P1
- 建议方案：增加“模型说明/为什么这样判断”展示层，说明输入行为、特征快照、四维状态、最终输出之间关系。
- 影响页面/模块：
  - `frontend/src/components/SessionReportPage.tsx`
  - 可选 `frontend/src/components/StatusPanel.tsx`
  - 答辩材料
- 建议负责 agent：`Agent E` + `Agent C` + `Agent F`
- 当前处理结论：记录，不进入热修。

### ISSUE-007 当前页面不足以支撑完整学习活动
- 来源：体验反馈补充
- 描述：用户认为当前页面上难以展开一轮完整学习活动并吸收反馈，因为学习需要参考对象和依据。
- 分类：产品逻辑问题
- 是否影响当前 MVP 演示：是
- 优先级：P0
- 建议方案：将执行页定位明确为“带参考区的学习工作台”，补足上下文，而不是把问题解释成单纯视觉问题。
- 影响页面/模块：
  - 执行页整体表达
  - 任务区 / 参考区 / 状态区的角色定义
- 建议负责 agent：`Agent D`，结构设计由总控 PM 先收口
- 当前处理结论：已完成最小 P0 修复。执行页通过任务绑定的学习参考区补齐学习上下文，页面定位已从“编码工作台”提升为“带参考区的学习工作台”。详见 `docs/热修记录-reference-zone.md`。
