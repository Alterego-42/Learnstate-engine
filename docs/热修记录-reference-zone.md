# 热修记录-reference-zone

## 背景
- 目标页面：执行页
- 负责分支：`codex/northstar-reference-zone-p0`
- 对应问题：`ISSUE-001`、`ISSUE-007`
- 热修目标：在不改执行页主结构的前提下，补齐学习资料参考区，让页面从“编码工作台”升级为“学习工作台”。

## 方案
- 把参考区放进任务卡内部，位于任务说明与编码区之间，默认展开、可手动收起。
- 参考区只保留三块最小可用内容：
  - `题目目标`：题意重述，帮助先对齐输入/输出与正确性要求
  - `思路框架`：从朴素法到优化法的推进线索
  - `学习步骤`：按“先理解 → 先写朴素法 → 再优化 → 再自测”的节奏组织练习
- 当前内容与任务绑定，优先覆盖 `demo-two-sum`，并补齐 `demo-binary-search`。

## 事件
- 新增前端轻量事件：
  - `reference.open`
  - `reference.section_change`
  - `reference.scroll`
  - `reference.close`
- 事件通过现有 `useEventReporter` 与 `/api/events` 发出，不新增状态机改造或额外基础设施。
- `reference.*` 当前只用于行为留痕，不改变现有执行页到复盘页的主链路。

## 为什么这次改动足够
- 题意、知识点、解题框架和学习步骤都能在当前页直接查看，学习活动不再只剩“写代码”。
- 参考区被限制在任务卡中并支持收起，编辑器主工作区仍然保持稳定，不压垮 demo-ready 页面。
- 文案强调“练习任务”“学习参考区”“学习步骤”，弱化了工具感，更接近学习工作台。

## 验证
- `frontend` 目录执行 `npm run build`：通过
- 原有编辑、运行、提交、状态轮询、打开复盘页入口均保留
- 新增 `reference.*` 事件不影响现有核心事件上报与状态推理
